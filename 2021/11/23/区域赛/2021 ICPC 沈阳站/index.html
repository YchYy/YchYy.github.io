# F. Encoded Strings I

第二道

Wa 了一发没考虑全面，而且当时做麻烦了。

# 题意

给一字符串，找一前缀，前缀的每一个字母的价值为此字母最后一次出现的位置后不同字母个数 + 'a'。

例如 aacc 的前缀 a -> a, aa -> aa, aac -> bba, aacc -> bbaa;

找使其字典序最大。

# 分析

倒序找一遍即可，记一下该字母最后位置后的不同字母个数。

# B. Bitwise Exclusive-OR Sequence

第三道

当时没有思路。。。。。。。。。。。。。。。人傻了。

# 题意

给一个 n 和一个 m, m 个关系，i , j , x, 表示 ai ⊕ aj == x, 求 a1 + a2 + ... + an 的最小值。

# 分析

将价值拆成二进制的每一位， 对于 x 如果其二进制下第 k 位为 1 ，表示 ai 的第 k 位与 aj 的第 k 位相反，若为 0 ，则表示相同。

为零时 显然二者第 k 位均为 0 更小。

从未遍历过的 ai 开始 dfs 一遍看与其相同的有多少个以及与其不同的有多少个，显然少的赋为 1 价值更小。

还需判一下是否有冲突的，有就输出 -1 .

# J. Luggage Lock

第四道

卡了半天，到比赛结束也没调出来，结束后发现再把 0 改为 10 再 chekc 一遍就过了。。。

# 题意

一把有四个小于 10 的数的锁，一次转动一格，连着的可以一起转动， 给定开始数字和结束数字， 问到达的最小步数。

# 分析

我做的麻烦了一些，把他处理成了道路铺设问题。

感觉还是 bfs 简单。bfs 以 0000 为起点，处理 0000 到每个数字的最小步数，那么 abcd -> xyzn 的步数就相当于 0000 -> (x-a)(y-b)(z-c)(n-d) 的步数.
